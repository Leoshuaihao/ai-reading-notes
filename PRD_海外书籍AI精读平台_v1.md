## 二、背景与目标

### 2.1 产品背景

中文读者在阅读海外投资经典书籍时，面临三个核心痛点：

1. **翻译质量参差不齐**：现有中文译本存在术语混乱、长难句直译、文化语境丢失等问题，导致核心概念被误读
2. **缺少结构化导读**：市面上"听书/讲书"类产品偏泛读和娱乐化，缺少面向专业投资领域的深度精读产品
3. **学习闭环缺失**：读者读完一本书后缺乏"思考→讨论→内化"的闭环，知识留存率低

### 2.2 产品定位

**AI驱动的海外投资经典深度精读平台**。不做全文翻译，不做内容分发，而是以"精读笔记"为核心——通过AI辅助生成逐章深度解读，帮助中文读者真正理解海外投资大师的思想精髓。

### 2.3 核心差异化

| vs. 得到/樊登 | vs. Blinkist | vs. 通用AI工具（Kimi/Claude） |
|---------------|-------------|------------------------------|
| AI驱动而非个人IP，内容更新更快、覆盖面更广 | 深度而非速度，逐章精读而非15分钟摘要 | 结构化产品体验（书架、章节导航、AI对话），而非纯工具 |

### 2.4 业务目标

| 目标 | 衡量指标 | 时间范围 |
|------|----------|----------|
| 验证产品价值 | 首本书精读笔记完成率 > 40% | MVP上线后1个月 |
| 验证付费意愿 | 付费转化率 > 3% | MVP上线后3个月 |
| 验证内容模式 | 单本书从选书到发布 < 3天（AI辅助） | MVP开发期间 |

## 五、功能需求详述（EARS原则）

### F-01 书籍概览卡片

| 编号 | 类型 | 需求描述 |
|------|------|----------|
| F-01-U1 | Ubiquitous | 系统应始终在页面顶部展示书籍概览卡片，包含：中文书名（主标题）、英文书名（副标题）、作者、豆瓣评分、标签（投资经典/阅读难度/预计阅读时间/首版年份）、一句话简介、CTA按钮 |
| F-01-E1 | Event-driven | 当用户点击"开始精读"按钮时，系统应平滑滚动到导言章节，且预留60px导航栏空间不被遮挡 |
| F-01-S1 | State-driven | 当页面宽度 ≤ 768px 时，书籍概览卡片应由横向布局切换为纵向布局，封面图片缩小至100×144px |

### F-03 逐章精读内容

| 编号 | 类型 | 需求描述 |
|------|------|----------|
| F-03-U1 | Ubiquitous | 系统应为每章提供以下内容模块：章节定位、原文金句（1-3句，英文可折叠+中文翻译+重要性解读）、核心概念深度解读（15条原则逐条拆解）、文化背景补充、反方观点与辩论、中国市场关联、关键术语词典（中英对照+解读）、要点总结（3-5条）、思考题（2道） |
| F-03-E1 | Event-driven | 当用户展开某章完整内容后，系统应将展开状态记录到浏览历史中，用户下次访问时保持展开状态 |
| F-03-U2 | Ubiquitous | 原文金句的英文原文应默认折叠（使用HTML details/summary标签），点击"📖 原文（点击展开）"后显示 |
| F-03-U3 | Ubiquitous | 术语词典应以中文为第一展示语言，英文以小字附后 |

### F-04 折叠/展开

| 编号 | 类型 | 需求描述 |
|------|------|----------|
| F-04-U1 | Ubiquitous | 系统应在每章默认状态下仅展示"章节定位"模块，其余内容（核心概念、文化背景、反方观点、中国市场关联、术语词典、要点总结、思考题）折叠隐藏 |
| F-04-E1 | Event-driven | 当用户点击"展开完整精读"按钮时，系统应以动画过渡方式展示该章全部内容，按钮文字变为"收起完整精读"，箭头旋转180° |
| F-04-E2 | Event-driven | 当用户点击"收起完整精读"按钮时，系统应以动画过渡方式折叠该章额外内容，按钮文字恢复为"展开完整精读" |
| F-04-S1 | State-driven | 当页面宽度 ≤ 768px 时，展开/收起按钮应占满整行宽度 |
| F-04-U2 | Ubiquitous | 每章的折叠区域上方应显示预览提示文字："点击展开：包含核心概念解读、文化背景、反方观点、中国市场关联、术语词典、思考题" |

### F-05 AI对话讨论

| 编号 | 类型 | 需求描述 |
|------|------|----------|
| F-05-U1 | Ubiquitous | 系统应在每道思考题下方提供AI对话入口，初始状态为折叠，显示"💬 跟AI讨论这道题" |
| F-05-E1 | Event-driven | 当用户点击对话入口时，系统应展开对话面板，展示预设的苏格拉底式引导对话（AI先问用户想法→用户回应→AI引用原文追问→用户深入→AI总结引导） |
| F-05-E2 | Event-driven | 当用户在输入框输入文字并点击发送（或按回车）时，系统应：①在对话区添加用户消息气泡（右对齐、橙色背景）②显示"AI正在思考..."打字指示器③1.5-2.5秒后添加AI回复气泡（左对齐、白色背景） |
| F-05-E3 | Event-driven | 当AI回复中引用书中原文时，系统应以金色左边框+米色背景的引用块展示，标注"📖"图标 |
| F-05-U2 | Ubiquitous | AI对话应遵循苏格拉底式引导原则：不直接给答案，先追问用户想法，引用书中原文，引导用户自己得出结论 |
| F-05-U3 | Ubiquitous | AI应知道当前对话所处的书籍、章节和题目上下文 |

### F-06 顶部导航栏

| 编号 | 类型 | 需求描述 |
|------|------|----------|
| F-06-U1 | Ubiquitous | 系统应在页面顶部固定显示导航栏（sticky定位），高度56px，包含Logo和章节链接列表 |
| F-06-E1 | Event-driven | 当用户点击导航栏中某章链接时，系统应阻止默认跳转行为，改为平滑滚动到对应章节，并预留60px偏移量避免被导航栏遮挡 |
| F-06-E2 | Event-driven | 当用户滚动页面时，系统应自动高亮当前可视区域内最靠上章节对应的导航链接 |
| F-06-S1 | State-driven | 当页面宽度 ≤ 768px 时，导航链接列表应支持横向滚动 |

### F-08 锚点分享

| 编号 | 类型 | 需求描述 |
|------|------|----------|
| F-08-E1 | Event-driven | 当用户滚动到某章节时，系统应使用 history.replaceState 更新浏览器URL hash为该章节的锚点ID，不触发页面滚动 |
| F-08-E2 | Event-driven | 当用户通过带锚点的URL访问页面时，系统应自动滚动到对应章节 |

## 六、流程说明

### 6.1 用户核心阅读流程

```
进入页面 → 浏览书籍概览卡片（3秒判断是否感兴趣）
         → 查看"为什么值得读"（4个理由卡片）
         → 查看阅读路线图（了解全书结构）
         → 点击"开始精读"或导航栏跳转到目标章节
         → 阅读章节定位（了解本章在全书中解决什么问题）
         → 展开完整精读（点击按钮）
         → 逐模块阅读（金句→概念→背景→反方观点→中国市场→术语→要点→思考题）
         → 点击AI对话讨论思考题
         → 输入自己的想法 → AI引导讨论
         → 继续下一章或结束阅读
```

### 6.2 内容生产流程

```
选书 → AI辅助提取章节结构和关键段落
    → AI生成初版精读笔记（概念解读、术语提取、练习题生成）
    → 人工编辑校对（选题判断、质量把关、语境补充、观点校正）
    → 发布
```

**AI负责**：初译、摘要生成、术语提取、练习题生成（效率层）  
**人负责**：选题判断、质量把关、语境补充、观点校正（质量层）

## 八、数据指标

### 8.1 核心指标

| 指标 | 定义 | 目标值 | 埋点方式 |
|------|------|:--:|----------|
| **章节展开率** | 点击"展开完整精读"的用户数 / 到达章节的用户数 | > 50% | 点击事件埋点 |
| **完读率** | 阅读到最后一章的用户数 / 进入页面的用户数 | > 25% | 滚动深度埋点 |
| **AI对话参与率** | 至少与AI对话一轮的用户数 / 看到思考题的用户数 | > 15% | AI对话发送事件埋点 |
| **页面停留时长** | 用户在精读笔记页面的平均停留时间 | > 8分钟 | 页面时长统计 |
| **分享率** | 通过锚点链接分享的次数 / 总访问次数 | > 5% | URL hash变更统计 |

### 8.2 辅助指标

| 指标 | 定义 |
|------|------|
| **跳出率** | 进入页面后5秒内离开的用户占比 |
| **导航点击分布** | 各章导航链接的点击次数分布 |
| **折叠章节分布** | 哪些章节的展开率最高/最低 |
| **移动端占比** | 移动端访问占比 |

## 九、验收标准

### 9.1 功能验收

| 编号 | 验收项 | 验收标准 | 验证方式 |
|------|--------|----------|----------|
| AC-01 | 书籍概览卡片 | 展示完整，标签≥4个，CTA按钮可点击跳转 | 手动+自动化 |
| AC-02 | 导航跳转 | 所有导航链接（11个）点击后平滑滚动到对应章节，不被导航栏遮挡 | 自动化 |

## 十、边界场景与异常处理

| 编号 | 场景 | 处理方式 |
|------|------|----------|
| EX-01 | 用户在移动端（375px）展开某章，内容过长 | 折叠区域设置 max-height 过渡动画，内容可正常滚动 |
| EX-02 | 用户展开多个章节后页面性能下降 | 折叠时使用 `display:none` 移除DOM渲染，展开时恢复 |
| EX-03 | AI对话API超时（>5秒） | 显示"AI正在思考..."超过5秒后提示"响应较慢，请稍候或稍后再试" |
| EX-04 | AI对话API返回错误 | 显示"AI暂时无法回应，请稍后再试"，保留用户已输入内容 |
| EX-05 | 浏览器不支持平滑滚动 | 降级为即时跳转（behavior: 'instant'） |
| EX-06 | URL hash指向不存在的章节 | 静默忽略，保持在页面顶部 |
| EX-07 | 用户通过浏览器前进/后退切换章节 | 监听 popstate 事件，平滑滚动到对应章节 |
| EX-08 | 浮动导航与顶部导航链接冲突 | 两者使用独立的事件监听，均支持平滑滚动 |
| EX-09 | 金句原文过长（>300字符） | 限制折叠区最大高度200px，超出部分滚动显示 |
| EX-10 | 网络断开时AI对话 | 输入框保持可用但发送按钮显示"离线"状态，点击后提示"请检查网络连接" |
| EX-11 | AI API成本失控（恶意刷量） | 前端基础频率限制（同设备10秒内最多5条）；后端API代理层IP级别限流（每分钟最多30条）；免费用户每日20轮上限 |
| EX-12 | 用户快速连续点击发送按钮 | 发送后按钮立即disabled，等待AI回复完成后恢复；前端debounce 1秒 |
| EX-13 | AI对话历史过长（>50轮） | 只保留最近30轮对话在localStorage中，超出部分自动裁剪；对话面板max-height 400px，超出滚动 |
| EX-14 | 用户复制内容未授权转载 | 复制时自动附加版权声明："—— 内容来源：AI精读笔记"；不阻止复制但保留溯源信息 |
| EX-15 | 多本书之间的状态隔离 | 不同书籍的展开状态、AI对话历史使用不同的localStorage key（如 `book_{id}_expanded`），防止状态混淆 |
| EX-16 | iOS Safari橡皮筋效果与sticky导航冲突 | 使用 `position: sticky` 而非 `position: fixed` 的模拟方案；或添加 `overscroll-behavior: none` 到导航容器 |
| EX-17 | Android Chrome地址栏显示/隐藏 | 地址栏的显示/隐藏会触发resize事件，需debounce处理，避免错误触发scroll相关逻辑 |
| EX-18 | 页面被iframe嵌入（点击劫持风险） | 添加 `X-Frame-Options: DENY` HTTP头；或前端检测 `window.self !== window.top` 时拒绝渲染 |
| EX-19 | AI返回空响应 | 显示"AI暂时无法生成回复，请换个角度再问一次"，不保留空白消息气泡 |
| EX-20 | 用户设备存储空间不足（localStorage quota exceeded） | 捕获 `QuotaExceededError`，降级为不持久化展开状态和对话历史，提示用户"存储空间不足，部分状态无法保存" |

## 十二、技术建议

### 12.1 推荐技术栈

| 层 | 技术选择 | 理由 |
|------|----------|------|
| **前端框架** | 纯HTML/CSS/JS（MVP）→ React/Vue（正式版） | MVP用原生最快上线，后续可平滑迁移 |
| **内容管理** | Markdown + 静态生成 | 精读笔记内容结构化，适合版本管理 |
| **AI对话** | OpenAI API / Claude API / 国产大模型 | 需支持长上下文（携带书籍章节信息） |
| **部署** | CloudFlare Pages / Vercel / 腾讯云静态托管 | 静态HTML即可部署，成本极低 |
| **埋点** | Umami / Plausible（自部署） | 隐私友好，数据自主可控 |

### 12.2 AI对话Prompt设计建议

```
你是一位投资导师，扮演Philip A. Fisher（《怎样选择成长股》作者）。
你正在与一位中文读者讨论书中的思考题。

原则：
1. 先用苏格拉底式提问引导读者说出自己的想法，不要直接给答案
2. 当读者表达观点后，引用书中的相关原文来深化讨论
3. 使用通俗的中文，偶尔保留关键英文术语
4. 语气温和但有挑战性——像一位严厉但关心你的导师
5. 每次回复控制在150字以内，聚焦一个点

当前上下文：
- 书籍：《怎样选择成长股》by Philip Fisher
- 章节：{chapter_title}
- 思考题：{question}
```
