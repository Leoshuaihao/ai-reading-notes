## 五、技术架构建议（研发评审核心结论）

### 5.1 推荐的MVP技术架构

```
┌─────────────────────────────────────────┐
│  Cloudflare Pages（免费静态部署+CDN）    │
├─────────────────────────────────────────┤
│  HTML + Alpine.js + htmx（增量25KB）     │
│  └─ 替代手写DOM操作                       │
│  └─ 原生支持SSE流式输出                   │
├─────────────────────────────────────────┤
│  Cloudflare Workers（API代理层）          │
│  └─ 转发AI API请求                       │
│  └─ API Key管理（环境变量）               │
│  └─ 基础速率限制                          │
│  └─ 流式SSE响应                           │
├─────────────────────────────────────────┤
│  Markdown + 11ty 静态生成（内容管理）     │
│  └─ Git版本管理                           │
│  └─ 非技术人员可编辑                      │
└─────────────────────────────────────────┘
```

### 5.2 关键技术决策

| 决策点 | 建议 | 理由 |
|--------|------|------|
| 折叠动画 | JS动态计算 `scrollHeight`，不使用硬编码 `max-height` | 避免长内容截断，动画更流畅 |
| 滚动监听 | `IntersectionObserver` 替代 scroll事件 | 消除强制重排，性能提升10倍+ |
| AI对话 | Cloudflare Workers代理 | API Key安全，支持流式输出，免费额度够用 |
| 渲染优化 | `content-visibility: auto` | 折叠内容不渲染，15条原则场景性能大幅提升 |
| 代码组织 | Alpine.js + htmx | 比React轻40倍，与现有HTML开发体验一致 |

## 六、补充验收标准（测试评审核心结论）

| # | 新增验收项 | 验收标准 |
|---|-----------|----------|
| AC-17 | AI对话安全 | 用户输入 `<script>alert(1)</script>` 被正确转义；输入敏感词后AI正确拒答 |
| AC-18 | AI对话频率限制 | 同一用户10秒内发送超过5条消息时提示"发送太快，请稍候" |
| AC-19 | 无障碍访问 | Tab键可导航所有交互元素，章节标题使用正确的heading层级 |
| AC-20 | 加载性能 | FCP < 1.5s, LCP < 2.5s, TTI < 3s |
| AC-21 | 社交分享 | 微信/微博分享时正确展示OG标题、描述和缩略图 |
| AC-22 | 展开状态持久化 | localStorage不可用时（Safari隐私模式）降级为每次访问默认折叠 |
| AC-23 | 跨浏览器兼容 | 新增微信内置浏览器、QQ浏览器、UC浏览器的兼容性验证 |
| AC-24 | 移动端键盘适配 | AI对话输入框在软键盘弹出时不被遮挡，自动滚动到最新消息 |
| AC-25 | 内容一致性 | 术语词典中同一英文术语在所有章节中翻译一致 |
