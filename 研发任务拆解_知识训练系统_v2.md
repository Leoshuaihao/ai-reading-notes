## 二、整体架构：5 块积木

```

                  ┌──────────────────────┐
                  │  BOOK_REGISTRY        │  19本书元数据
                  │  CONCEPTS             │  12个概念定义

        ┌────────────┼────┼────┼────┼──────────────┐
        │            │    │    │    │              │
    ┌───▼───┐  ┌─────▼──┐ ┌▼────▼─┐ ┌─────▼──┐ ┌──▼──────┐
    │积木①  │  │积木②   │ │积木③  │ │积木④   │ │积木⑤   │
    │知识题库│──▶│训练引擎│ │自测系统│ │能力面板│ │概念联动 │
    └───────┘  └───┬────┘ └──┬───┬┘ └────┬───┘ └────┬────┘
                   │         │   │       │          │
                   └────┬────┘   │       │          │
                        │    读取进度   │          │
                        └──────┬───────┘          │
                               └──────────────────┘
                               localStorage ability_profile
```

**数据流向**：
- 积木①写入 `data_v2.js KNOWLEDGE_POINTS`（只读数据，不含进度）
- 积木②读取 ① 的数据，写入 `localStorage ability_profile`
- 积木③读取 ① 的数据生成题，读取章节内容生成解析
- 积木④读取 `ability_profile` 渲染面板
- 积木⑤读取 `ability_profile` 标记概念状态

**依赖关系**：① → ② → ④/⑤（串行），③ 与 ①/② 可并行

## 三、积木①：知识题库

### 目标

### 交付物
1. `data_v2.js` 中新增 17-27 个 `KNOWLEDGE_POINTS` 条目
2. 每知识点包含以下字段：

```javascript
{
  id: 'owner-earnings',           // 唯一ID
  name: '股东盈余',                // 中文名
  emoji: '💰',                    // 图标
  domain: 'valuation',            // 所属域
  level: 1,                       // 难度 1-5
  description: '掌握...',          // 一句话描述
  formula: '股东盈余 = 净利润 + 折旧摊销 - 维持性资本支出',  // 核心公式
  formulaDisplay: '股东盈余 = <b>净利润</b> + 折旧摊销 - <b>维持性资本支出</b>',  // 格式化展示
  conceptKeyPoints: [             // 3-5条核心要点
    '净利润≠经济利润',
    '折旧是"非现金支出"，需要加回',
    '维持性资本支出是保持竞争力的必要投入'
  ],
  chapters: {                     // 多书关联
    'buffett-letters': {
      ch: [1, 5],
      ref: '第1章「内在价值」、第5章「收购与估值」',
      usageNote: '详细说明该知识点在这本书中如何使用（50-100字）'
    }
  },
  training: {                     // 训练素材
    guided: {                     // 引导层：手把手教学
      intro: '假设一家公司...',
      steps: [
        { label: '步骤1：找净利润', formula: '净利润 = 归母净利润' },
        { label: '步骤2：加回折旧', formula: '折旧摊销 = ...' }
      ],
      targetAnswer: 12500000000   // 标准答案（数字）
    },
    practice: {                   // 练习层：独立计算
      scenario: '某公司年报数据如下：...',
      data: { revenue: 1000, cost: 600 },
      questions: [
        { q: '请计算该公司的所有者收益', answer: 450, hint: '...' }
      ]
    },
    realWorld: {                  // 实战层：真实案例
      company: 'apple',           // 引用 COMPANY_CASES 中的 key
      analysis: 'Apple 的 maintenance capex 如何区分...'
    }
  }
}
```

### 知识点覆盖计划

按 5 域分配，每域 4-6 个核心知识点：

| 域 | 现有 | 新增 | 知识点方向 |
|------|------|------|-----------|
| **valuation** 资产估值 | 2 | +3 | PE/PB/ROE拆解、FCF计算、终值估算 |
| **business** 商业认知 | 0 | +5 | 护城河识别、波特五力、管理层评估、竞争优势持续性、规模效应 |
| **uncertainty** 风险 | 0 | +5 | 概率思维、凯利公式、期望值、肥尾风险、杠铃策略 |
| **cycle** 周期 | 0 | +4 | 信贷周期指标、估值水位判断、极度恐惧指标、三周期共振 |
| **mind** 心性 | 0 | +5 | 认知偏差清单、独立判断训练、逆向思维检验、延迟满足量化、误判心理 |

**总计目标：28 个知识点**

## 四、积木②：训练引擎

### 目标
完成 training.js 的 4 层训练流（概念层→引导层→练习层→实战层），每层可独立验证。

### 依赖
积木①完成后，KNOWLEDGE_POINTS 有完整 training 字段。

### 交付物
1. `assets/js/training.js` 完整实现

### 4 层训练流设计

```
概念层 → 引导层 → 练习层 → 实战层 → 完成
  │         │        │        │
  │         │        │        └── 真实公司案例分析 + 开放式问答
  │         │        └── 独立计算/判断 + 答案校验（数值/段落匹配）
  │         └── 手把手步骤教学 + 示范答案（用户只需跟随）
  └── 公式展示 + 要点解释 + "我已理解"按钮（knowledge check）
```

### 各层实现要点

**概念层**（已有骨架，需补数据绑定）
- 显示 `formulaDisplay` + `conceptKeyPoints` + `description`
- 按钮"我已理解"写入 `ability_profile`

**引导层**（需新建）
- 从 `kp.training.guided` 读取步骤
- 逐步展示，用户每步点"下一步"
- 每步显示公式/计算过程
- 最后展示标准答案供对比

**练习层**（需新建）
- 从 `kp.training.practice` 读取题目
- 输入框（数字题）或文本区（开放题）
- 答案校验：数字题匹配 ±5% 容差，文字题关键词匹配
- 错误时显示 hint，允许重试 3 次
- 答题记录写入 `ability_profile`

**实战层**（需新建）
- 从 `kp.training.realWorld` 读取案例
- 展示公司背景 + 数据
- 开放问题 + AI 点评（可选，调用 chat.js）
- 完成标记

### 进度持久化

```javascript
// ability_profile 结构（localStorage）
{
  version: 2,
  updated: "2026-07-21T20:00:00Z",
  domains: {
    valuation: { score: 65, completed: 2, total: 5 },
    business:  { score: 0,  completed: 0, total: 5 },
    ...
  },
  knowledgePoints: {
    'owner-earnings': {
      levels: { concept: true, guided: true, practice: false, real_world: false },
      attempts: 3,
      completedAt: "2026-07-21T20:00:00Z"
    }
  }
}
```

### 验收标准
- [ ] 4 层训练流完整可走通
- [ ] 引导层步骤展示正确
- [ ] 练习层数字答案容差校验 ≤5%
- [ ] localStorage 写入/读取正确
- [ ] 从任意书页 `.chapter-card` 可触发 Modal

## 五、积木③：自测系统

### 目标
为每章底部嵌入 1-2 道自测题，用户读完章节后即时检验理解。

### 依赖
积木①的知识点定义（题目可基于知识点生成），或直接基于章节内容。

### 交付物
1. 自测题数据（嵌入 `data_v2.js` `CHAPTER_QUIZZES` 或独立 JSON）
2. `assets/js/quiz.js` — 自测渲染 + 校验引擎
3. 所有 19 本书章节页注入 `self-quiz` 块

### 题目类型

| 类型 | 示例 | 校验方式 |
|------|------|---------|
| 单选 | "安全边际的公式是？" | 4 选项，选对即过 |
| 填空/计算 | "折现率 10%，永续增长 3%，终值乘数 = 1/(0.1-___)" | 数字匹配 ±5% |
| 判断 | "股价越低，风险越小。（对/错）" | 对/错 |
| 配对 | "将左侧概念与右侧大师配对" | 全对即过 |

### 嵌入位置
每章 `.chapter-card` 的 `takeaway-list` 之后、`exercise-box` 之前，作为折叠卡片：

```html
<div class="self-quiz" data-kp-id="owner-earnings">
  <div class="quiz-header" onclick="toggleQuiz(this)">
    自测 · 股东盈余
  </div>
  <div class="quiz-body" hidden>
    <div class="quiz-question">...</div>
    <div class="quiz-options">...</div>
    <div class="quiz-feedback" hidden>✅ 正确！</div>
  </div>
</div>
```

### 生成策略
- 每本书每章 1 题 = ~370 题（19 本 × 平均 19.5 章）
- 题目关联到最近的知识点（`data-kp-id`），方便跳转训练场
- 用 DeepSeek 批量生成题目 JSON → 脚本注入 HTML

### 验收标准
- [ ] 每章至少 1 道自测题
- [ ] 单选/判断/计算 3 种题型至少各 30 题
- [ ] 选题后即时反馈（对/错 + 解释）
- [ ] 不影响已有章节折叠逻辑

## 九、接口契约（agent 之间的协议）

所有 agent 读写同一个 `data_v2.js`，必须遵守以下约定：

### A. KNOWLEDGE_POINTS 追加规则
- 新条目追加到现有数组末尾
- `id` 必须全局唯一，命名规范：`{concept}-{aspect}`，如 `owner-earnings`、`moat-identification`
- 不删除、不重排已有条目

### B. localStorage 读写规则
- 只用 `ability_profile` 这一个 key（已定义）
- 写入前先 `JSON.parse` 读取现有数据，merge 后写回
- 格式见第六节

### C. 文件修改范围
| Agent | 修改文件 |
|-------|---------|
| 积木① | `assets/js/data_v2.js`（追加 KNOWLEDGE_POINTS） |
| 积木② | `assets/js/training.js`（全覆盖） |
| 积木③ | `assets/js/quiz.js`（新建）+ 19本书 HTML（追加 self-quiz 块）或 `gen_books_v2.py`（模板注入） |
| 积木④ | `my.html`（新增面板区域）+ `assets/css/practice.css`（新增雷达图样式） |
| 积木⑤ | `concepts.html`（新增状态逻辑） |

### D. 冲突避免
- 积木②/③/④/⑤ 都只读 KNOWLEDGE_POINTS，不写
- 积木① 只追加 KNOWLEDGE_POINTS，不碰 HTML/CSS
- 积木④/⑤ 只读 ability_profile，不写
- 唯一写入 ability_profile 的是积木②（训练引擎）
