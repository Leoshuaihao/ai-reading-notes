## 1. 背景

### 1.1 V1 做了什么

V1 完成了从 0 到 1 的 MVP：将 6 本投资经典书籍转化为中文精读笔记网页，每章一个折叠卡片，包含核心概念解读、金句、文化背景、中国市场关联等。部署在 GitHub Pages，纯静态架构。

### 1.2 V1.5（当前线上版本）额外做了什么

在 V1 基础上，本地模型自主完成了以下进化：

| 维度 | V1 | V1.5（当前） |
|------|-----|-------------|
| 书籍数量 | 6 本 | **16 本** |
| 精读章节 | ~86 章 | **278 章** |
| 独立页面 | 2 个 | **6 个** |
| 覆盖大师 | 4 位 | **8+ 位** |
| 体系化 | 无 | **五域修行体系** |
| 游戏化 | 无 | **9 级等级 + 成就 + XP** |
| 可视化 | 无 | **能力雷达图 + 登山进度 + 金字塔** |
| 跨书连接 | 无 | **概念地图 + 跨山连接** |
| 用户互动 | AI 对话 | **+ 趣味测试 + 今日精读 + 金句墙** |

### 1.3 为什么需要 V2 PRD

V1.5 的功能是"自发进化"出来的，缺乏系统性规划。V2 需要：
- 基于竞品调研做有依据的产品决策
- 补齐 V1.5 的短板
- 为下一阶段增长建立明确的产品路线图

## 2. 竞品与市场分析

### 2.1 核心发现

中文市场存在 **三重空白**：

1. **"中文版 Blinkist for 投资书籍"完全空白** — 海外有 Blinkist/Shortform 等书籍摘要平台，但无中文投资专精版本
2. **"精读笔记 + 知识图谱 + 游戏化进阶"交叉领域无人占据** — 现有方案要么是个人博客笔记，要么是碎片化社区帖，要么是纯课程平台
3. **以经典投资书籍为锚点的结构化学习路径无先例** — 即使有学习路径产品（如 vi_system），也脱离经典书籍文本

### 2.2 竞品矩阵

| 竞品 | 类型 | 优势 | 劣势 | 我们的差异化 |
|------|------|------|------|------------|
| 小敏笔记 | 个人博客 | 原创、免费 | 内容少、无体系、无游戏化 | 16本书体系 + 五域修行 |
| 投资导航 | 链接聚合 | 有积分体系 | 非深度精读、内容浅 | 278章深度精读 |
| Blinkist | 海外书籍摘要 | 量大、质量好 | 无中文、无投资专精 | 中文原生 + 投资专精 |
| vi_system | 学习路径 | 模块化设计好 | 内容极简、无书籍关联 | 书籍为锚点的体系 |
| 雪球/知乎 | 碎片社区 | 内容丰富 | 不成产品、无结构 | 产品化的完整体验 |

### 2.3 我们的差异化定位

> **"中文市场唯一以经典投资书籍为锚点、以体系化学习路径为核心、以游戏化机制为激励的投资学习平台。"**

## 3. 目标用户与使用场景

### 3.1 用户画像

| 画像 | 描述 | 核心需求 |
|------|------|---------|
| **投资小白** | 刚接触投资，不知道从哪本书开始 | 入门引导、书单推荐、降低认知门槛 |
| **进阶学习者** | 读过一些书，但不成体系 | 体系化路径、知识串联、查漏补缺 |
| **深度研究者** | 想深入理解某位大师/某个概念 | 跨书对比、概念地图、原文精读 |
| **产品/创业者** | 想用投资思维提升商业判断 | 商业认知域、实用原则、案例关联 |

### 3.2 核心场景

| 场景 | 用户行为 | 平台响应 |
|------|---------|---------|
| 不知道该读什么 | 打开修行首页 | 看到五座山 + 趣味测试推荐 |
| 碎片时间学习 | 点击今日精读 | 每日一章随机推荐 |
| 想建立知识体系 | 按域登山 | 从 Lv.1 到 Lv.4 递进阅读 |
| 想跨书理解一个概念 | 打开概念地图 | 看到 12 个概念的多维视角 |
| 想查看学习成果 | 打开个人中心 | 看到等级/成就/能力雷达/登山进度 |
| 遇到投资问题 | 打开诊断引擎 | 用大师框架分析具体股票 |

## 4. 产品架构（V2 目标态）

### 4.1 信息架构

```
🏠 修行首页
├── 🎯 能力画像（雷达图 + 等级 + XP）
├── ⛰️ 五域体系（5张域卡片，每张含书籍+进度+跨山连接）
├── 💬 金句滚动墙
├── 🧩 趣味测试
├── 📅 今日精读
└── 📊 修行总览

📚 完整书架 → 16本书卡片 → 进入精读
🕸️ 概念地图 → 12个概念 × 多维度展开
🧘 个人中心 → 等级/成就/能力树/登山图/阅读进度
🔍 诊断引擎 → 股票分析（Fisher + Graham + Buffett框架）

📖 单书精读页（16个）
├── Hero + 作者简介
├── 核心框架/原则速览
├── 阅读路线图 + 进度条
├── 导言 + N章折叠卡片
│   ├── 核心问题 + 关键概念
│   ├── 金句 + 文化背景
│   ├── 反方观点 + 中国市场
│   ├── 术语词典 + 思考题
│   └── 自测卡片 + 跨书对话
└── AI 对话组件
```

### 4.2 用户旅程

```
新用户 → 趣味测试 → 推荐第一本书 → 开始阅读
    ↓
读第一章 → XP +1 → 进度条更新 → 雷达图微调
    ↓
读完第一本 → 成就解锁 → 等级升级 → 解锁下一本书
    ↓
多域开启 → 跨山连接激活 → 概念地图可探索
    ↓
读完多本 → 体系形成 → 诊断引擎可用 → 实战检验
```

## 5. V2 功能清单

### 5.1 P0 — 补齐当前短板（必须做）

| ID | 功能 | EARS 需求描述 | 价值 | 复杂度 |

| F1 | 新用户引导流程 | When 用户首次访问，the system shall 展示3步引导（选域→测水平→推第一本书） | 降低新用户流失 | 中 |
| F2 | 跨页面数据一致性 | The system shall 确保修行首页、书架、个人中心的阅读进度数据完全同步 | 当前 my.html 和 index.html 各自读 localStorage，需统一数据层 | 低 |

## 6. 技术演进方向

### 6.1 短期（V2.0）

- **数据层统一**：创建 `assets/js/data.js`，将 BOOK_META、DOMAINS、LEVELS 等静态配置抽离为共享模块，避免 index.html 和 my.html 重复维护
- **进度存储标准化**：统一 `reading_progress` 的读写接口，确保所有页面用同一套函数
- **构建工具引入**（可选）：如果页面数量继续增长，考虑用简单的构建脚本做 HTML include（导航栏、footer 复用）

### 6.2 中期（V2.5）

- **搜索功能**：纯静态搜索（Lunr.js 或 Fuse.js），支持跨书搜索概念、金句、章节
- **个性化推荐**：基于阅读历史的协同过滤或简单规则引擎
- **PWA 化**：Service Worker 缓存，离线可用

### 6.3 长期（V3.0）

- **用户系统**：如果要做社区/评论/云端同步，需要引入轻量后端（如 Supabase）
- **内容平台化**：允许用户上传自己的读书笔记，形成 UGC + PGC 混合模式

## 9. 风险与依赖

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 纯静态架构限制 | 用户系统、评论、云端同步无法实现 | V3.0 前保持静态，评估 Supabase |
| 内容生产速度 | 新书依赖人工+AI 协作，难以规模化 | 完善 ADD_BOOK_STANDARD.md，提升 AI 自动化程度 |
| localStorage 数据丢失 | 用户清除浏览器数据后进度丢失 | 短期提醒用户，长期考虑云端备份 |
| GitHub Pages 访问速度 | 国内用户可能较慢 | 考虑 Cloudflare Pages 或国内 CDN |

## 10. 附录

### 10.1 竞品调研详细记录

见独立调研报告（Agent 调研输出），核心结论：

- 直接竞品：无（中文市场空白）
- 间接竞品：小敏笔记（个人博客）、投资导航（链接聚合）、Blinkist（海外英文）、vi_system（学习路径 Demo）
- 差异化窗口：书籍锚点 + 体系路径 + 游戏化激励 的三合一模式

### 10.2 V1.5 自发进化分析

V1.5 的功能演进虽然是"自发"的，但方向高度正确——从"内容展示"到"体系化学习"再到"游戏化激励"，恰好是教育类产品的标准进化路径。V2 需要在这个方向上做系统性加固，而非另起炉灶。
