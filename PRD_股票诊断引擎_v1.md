# 股票诊断引擎 PRD

> **版本**：v1.0  
> **日期**：2026-07-16  
> **关联**：精读笔记《怎样选择成长股》第 3 章 15 条原则  
> **架构原则**：数据层大师无关 + 引擎层插件化 + 不编数据、不推荐买卖

---

## 一、产品定位

**用 Fisher 的眼睛看公司。** 用户输入一只股票，系统基于 15 条原则逐条分析，生成一份"Fisher 会怎么看这家公司"的诊断报告。

不是荐股工具，是**学习方法论的工具**。

## 三、15 条原则评分设计

每条原则按三类处理：

| 处理类型 | 说明 | 适用原则 |
|:--:|------|------|
| **可量化** | 公开数据可直接计算 | 1/5/6/10/13 |
| **半量化** | 有公开指标但需结合判断 | 2/3/4/11/12/14 |
| **需调研** | 公开数据无法覆盖，必须 Scuttlebutt | 7/8/9/15 |

### 逐条评分逻辑

#### 原则 1：市场潜力 — "足够大的市场空间支撑至少数年增长"

| 评分 | 条件 |
|:--:|------|
| 9-10 | 行业 3 年 CAGR > 15% 且公司份额在扩大 |
| 7-8 | 行业增速 5-15%，公司是龙头 |
| 5-6 | 行业增速 0-5%，公司份额稳定 |
| 3-4 | 行业负增长，但公司在抢份额 |
| 1-2 | 行业和公司都在萎缩 |

**数据来源**：行业研报增速 + 公司营收 3 年 CAGR（公开财报）

#### 原则 2：管理层决心 — "现有产品增长耗尽时，是否有决心开发新产品"

| 评分 | 条件 |
|:--:|------|
| 9-10 | 研发费用率 > 行业均值 + 有明确的第二曲线布局 |
| 7-8 | 研发费用率高于行业中位数 |
| 5-6 | 研发费用率行业中位数 |
| 3-4 | 研发费用率低于行业 + 无新业务布局 |
| 1-2 | 几乎没有研发投入 |

**数据来源**：研发费用率（财报）+ 年报中"新业务/新产品"披露

#### 原则 3：研发效率 — "研发投入相对于规模是否有效"

| 评分 | 条件 |
|:--:|------|
| 9-10 | 每元研发投入产出比 > 行业均值 150% |
| 7-8 | 高于行业均值 |
| 5-6 | 行业均值附近 |
| 3-4 | 低于行业均值 |
| 1-2 | 显著低于行业均值 |

**数据来源**：研发费用 / 营收增量（近 3 年）（财报推算）

#### 原则 4：销售组织 — "是否有高于平均水平的销售团队"

| 评分 | 条件 |
|:--:|------|
| 9-10 | 人均创收 > 行业均值 150% + 销售费用率低于行业（效率高） |
| 7-8 | 人均创收高于行业均值 |
| 5-6 | 行业均值 |
| 3-4 | 低于行业均值 |
| 1-2 | 显著低于行业均值 |

**数据来源**：人均创收 = 营收/员工数 + 销售费用率（财报）

#### 原则 5：利润率 — "是否有有价值的利润率，趋势如何"

| 评分 | 条件 |
|:--:|------|
| 9-10 | 毛利率 > 行业均值 120% 且 3 年趋势稳定或上升 |
| 7-8 | 毛利率高于行业均值，趋势稳定 |
| 5-6 | 毛利率在行业均值附近 |
| 3-4 | 毛利率低于行业均值 |
| 1-2 | 毛利率低于行业均值且持续下降 |

**数据来源**：毛利率 3 年数据 + 行业均值（财报 + 行业对比）

#### 原则 6：利润率维护 — "在做什么来维持或改善利润率"

| 评分 | 条件 |
|:--:|------|
| 9-10 | 费用率 3 年下降 + 毛利率 3 年不降（效率提升型） |
| 7-8 | 费用率稳定 + 毛利率稳定 |
| 5-6 | 提价维持利润率（价格驱动型，有上限） |
| 3-4 | 费用率上升 + 毛利率下降 |
| 1-2 | 费用率失控 + 毛利率崩塌 |

**数据来源**：费用率 3 年趋势（财报）

#### 原则 7：劳动关系 — "是否有卓越的劳动关系"

> ⚠️ **公开数据无法覆盖。标注"需调研"，不编造评分。**

展示信息：
- 员工人数变化趋势（扩张 or 裁员）
- 人均薪酬 vs 行业（过高 or 过低都是风险）
- 如果有公开的员工评价（脉脉/Glassdoor），可引用但不评分

**输出**：📋 "Fisher 会说：去跟这家公司的前员工聊聊。问问他们为什么离开。"

#### 原则 8：高管关系 — "是否有卓越的高管关系"

> ⚠️ **公开数据无法覆盖。标注"需调研"。**

展示信息：
- 高管离职率（近 3 年）
- CEO/CFO 是否频繁更换
- 高管薪酬 vs 行业（极端值标注）

**输出**：📋 "Fisher 会说：看看高管之间是否相互尊重。频繁更换 CFO 是一个危险信号。"

#### 原则 9：管理层深度 — "管理层是否有足够的深度"

> ⚠️ **公开数据无法覆盖。标注"需调研"。**

展示信息：
- CEO 是否兼任董事长（权力过度集中）
- 高管团队平均任期
- 是否有明确的接班人计划披露

**输出**：📋 "Fisher 会说：如果 CEO 离职这家公司就瘫痪，那它不值得投资。"

#### 原则 10：成本分析 — "成本分析和会计控制有多好"

| 评分 | 条件 |
|:--:|------|
| 9-10 | 连续 5 年无财务重述 + 审计意见标准无保留 + 无关联交易异常 |
| 7-8 | 近 3 年无财务重述 + 标准审计意见 |
| 5-6 | 有少量财务重述但金额不重大 |
| 3-4 | 近 3 年有重大财务重述 |
| 1-2 | 审计意见非标 + 财务重述频繁 |

**数据来源**：审计报告 + 财务重述公告（公开信息）

#### 原则 11：竞争优势 — "是否有行业特有的竞争优势"

| 评分 | 条件 |
|:--:|------|
| 9-10 | 市场份额 > 30% 且 3 年稳定或上升 + 品牌/专利/网络效应明确 |
| 7-8 | 市场份额 > 15% + 有可识别的竞争壁垒 |
| 5-6 | 市场份额 5-15%，竞争壁垒模糊 |
| 3-4 | 市场份额 < 5%，无明显壁垒 |
| 1-2 | 完全竞争市场，无差异化 |

**数据来源**：市场份额（行业报告）+ 护城河分析（公开信息）

#### 原则 12：利润导向 — "短期还是长期的利润观"

| 评分 | 条件 |
|:--:|------|
| 9-10 | 研发+资本开支增速 > 利润增速（在投入未来） |
| 7-8 | 研发+资本开支增速 ≈ 利润增速 |
| 5-6 | 利润增速 > 投入增速（可能吃老本） |
| 3-4 | 削减研发/资本开支来维持利润 |
| 1-2 | 大规模裁员+削减投入+回购拉股价 |

**数据来源**：研发费用增速 vs 利润增速 + 资本开支增速（财报）

#### 原则 13：融资需求 — "增长是否需要大量股权融资稀释股东"

| 评分 | 条件 |
|:--:|------|
| 9-10 | 近 5 年无增发 + 自由现金流为正 |
| 7-8 | 近 3 年无增发 + 自由现金流基本平衡 |
| 5-6 | 有增发但总股本稀释 < 5%/年 |
| 3-4 | 频繁增发，股本稀释 > 5%/年 |
| 1-2 | 每年大规模增发，股本稀释 > 10%/年 |

**数据来源**：总股本变化 + 增发公告（公开信息）

#### 原则 14：沟通透明度 — "好事大谈坏事沉默"

| 评分 | 条件 |
|:--:|------|
| 9-10 | 业绩说明会如实回答尖锐问题 + 年报风险披露详实 |
| 7-8 | 沟通基本透明，偶尔回避敏感问题 |
| 5-6 | 报喜不报忧的倾向明显 |
| 3-4 | 经常回避负面问题 + 风险披露敷衍 |
| 1-2 | 被监管处罚信息披露违规 |

**数据来源**：交易所监管记录 + 年报风险披露篇幅 + 投资者关系评价

#### 原则 15：管理层诚信 — "是否有毋庸置疑的正直品格"

| 评分 | 条件 |
|:--:|------|
| 9-10 | 无关联交易异常 + 无大股东占款 + 无违规记录 + 分红/回购善待小股东 |
| 7-8 | 无重大违规 + 关联交易在合理范围 |
| 5-6 | 有轻微违规记录 |
| 3-4 | 关联交易频繁 + 大股东高比例质押 |
| 1-2 | 大股东占款/掏空上市公司/财务造假嫌疑 |

**数据来源**：交易所监管记录 + 关联交易公告 + 大股东质押比例 + 违规处罚记录

## 七、边界场景

| 场景 | 处理 |
|------|------|
| 用户输入不存在的股票 | 提示"未找到该股票，请检查代码" |
| 用户输入港股/美股 | MVP 阶段只支持 A 股，提示"即将支持港股/美股" |
| 某条原则数据完全缺失 | 标注"数据不足" + 说明原因 + 引导 Scuttlebutt |
| 公司上市不足 3 年 | 标注"上市时间不足，部分趋势分析无法进行" |
| 财报数据可能滞后 | 标注数据截止日期 |
| 同一用户频繁查询 | 每日限制 5 次（免费），防止 API 成本失控 |

## 八、验收标准

| 编号 | 验收项 | 标准 |
|------|--------|------|
| AC-D01 | 输入茅台 → 生成完整报告 | 15 条原则逐条展示，每条有评分或"需调研"标注 |
| AC-D02 | 每条可量化原则标注数据来源 | 至少包含"来源：2025年报"或"来源：XX行业报告" |
| AC-D03 | 需调研原则不编造评分 | 原则 7/8/9 显示"需调研"而非假分数 |
| AC-D04 | 免责声明可见 | 报告末尾有"不构成投资建议"声明 |
| AC-D05 | 报告可分享 | 生成唯一链接，可分享给他人查看 |
| AC-D06 | 多大师架构可扩展 | FisherEngine 实现 MasterEngine 接口 |
